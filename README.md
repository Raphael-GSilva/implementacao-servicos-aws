# Implementação de Serviços AWS para uma Farmácia

## Sobre o Projeto

Este projeto apresenta uma proposta de implementação de serviços AWS para uma farmácia fictícia, com foco na modernização da infraestrutura tecnológica, redução de custos operacionais e aumento da disponibilidade dos sistemas.

A solução foi desenvolvida como parte de um desafio prático da Formação AWS da DIO, aplicando conceitos de computação em nuvem em um cenário corporativo.

---

## Objetivo

Propor uma arquitetura baseada em serviços AWS capaz de substituir parte da infraestrutura local da empresa, proporcionando:

- Redução de custos com servidores físicos;
- Maior disponibilidade dos sistemas;
- Escalabilidade da infraestrutura;
- Segurança das informações;
- Facilidade de administração.

---

## Serviços Utilizados

### AWS EC2

Serviço utilizado para hospedar os sistemas internos da empresa.

**Aplicação**

Hospedagem do sistema de gestão da farmácia.

**Benefícios**

- Redução dos custos com infraestrutura física;
- Alta disponibilidade;
- Escalabilidade;
- Flexibilidade na utilização de recursos.

---

### AWS S3

Serviço utilizado para armazenamento de arquivos corporativos.

**Aplicação**

Armazenamento de documentos, backups, notas fiscais e imagens de produtos.

**Benefícios**

- Alta durabilidade;
- Backup automatizado;
- Redução de custos com armazenamento;
- Compartilhamento seguro de arquivos.

---

### AWS RDS

Serviço de banco de dados relacional gerenciado.

**Aplicação**

Armazenamento das informações de clientes, estoque, fornecedores e vendas.

**Benefícios**

- Backups automáticos;
- Alta disponibilidade;
- Escalabilidade;
- Redução da administração da infraestrutura.

---

## Benefícios da Solução

Com a implementação proposta espera-se:

- Redução dos custos operacionais;
- Modernização da infraestrutura;
- Maior disponibilidade dos sistemas;
- Segurança das informações;
- Escalabilidade conforme o crescimento da empresa.
